# 公文写作助手 — AI 辅助写作 SOP

## 触发条件

用户需要写以下类型材料时，加载此 skill 并按流程执行：

**10 种内建文体：**
1. 工作总结
2. 述职报告
3. 汇报材料
4. 讲话稿
5. 经验材料
6. 调研报告
7. 先进典型事迹
8. 工作思路/措施/方案类
9. 会议纪要
10. 新闻稿

**通用模板：** 上述未覆盖的其他规范性写作任务（通过头脑风暴+素材灵活构建提示词）

**进阶场景：** 用AI修改材料（整体提升/局部精修/风格转换）、去AI味

---

## 一、提示词结构

无论写什么材料，结构化提示词是基础。标准结构：

```
【角色】— 你扮演谁（身份+背景+能力）
【任务】— 你要写什么（材料类型+主题+字数）
【要求】— 怎么写（风格+结构+约束条件）
【素材】— 用什么写（原始数据/范文/方法论）
```

**P 的多种形态：**

| 形态 | 用法 | 目的 |
|------|------|------|
| **素材** | 政策数据、工作成果、调研发现 | 让 AI 基于真实信息创作 |
| **优质样例（正例）** | 完整范文或段落 | 让 AI 模仿语言风格、结构逻辑 |
| **反面样例（反例）** | 需要避免的写法 | 让 AI 规避常见问题 |
| **正反样例组合** | 同时给好和坏的案例 | 明确边界，效果更精准 |
| **方法论** | 写作规则/结构模板 | 让 AI 先学会规则再执行 |

**万能角色问法（用户不确定时）：**
> 直接问 AI：「你想让我扮演什么角色效果最好？」

## 二、六步写作流程

### Step 1：拆解写作任务 + 头脑风暴

写作开始前，先与用户充分沟通，收集完整信息。这一步不是走形式——问得越细，后面生成越准，返工越少。

#### 1.1 文体确认 + 素材收集

```
先问用户：
  要写什么文体？
    内建文体（10种）→ 加载对应模板
    其他文体 → 走通用模板流程
  有素材吗？
    有 → 让用户提供（文字/文档/数据）
    没有但有方向 → web_search 搜索补充
    有范文参考 → 让用户提供1-2篇
```

> 💡 素材比提示词技巧更重要。同样水平的提示词，素材扎实→产出扎实，素材空洞→产出空洞。

#### 1.2 头脑风暴（核心环节）

拿到文体和素材后，不要直接开写。先按以下维度与用户逐一聊，收集完整背景：

```
写作背景与用途
├─ 这是给谁看的？（领导/上级/内部/公开）
├─ 在什么场合用？（汇报/评选/宣传/存档）
├─ 写这份材料的直接原因是什么？
│  （上级要求/年度例行/问题整改/创新申报）

核心信息
├─ 你希望读者看完后知道什么、记住什么？
├─ 这次写作有没有特别的难点或敏感点？
├─ 之前写过类似的吗？效果怎么样？这次想改进什么？

作者与视角
├─ 以谁的身份写？（个人/部门/单位）
├─ 需要怎么样的语气？（正式严肃/生动感人/客观中立）

风格偏好
├─ 有没有你喜欢的范文或写法可以给我参考？
├─ 有没有你不想要的风格？（太官方/太口语/太夸张）
├─ 字数预期？（精确数字或大致范围）
```

> 这个环节不需要一次问完所有问题。先问关键的（用途+核心信息），让用户先给你一些内容，再根据反馈追问补充。

#### 1.3 确认与对齐

开始写作前，把理解复述给用户确认：

```
我理解你要写的是：
- 文体：[工作总结/述职报告/...]
- 身份：[XX部门XX职务]
- 字数：[约XXX字]
- 用途：[内部汇报/上级送审...]
- 核心素材：[已提供/待补充]
- 风格：[具体风格要求]

你看这样理解对吗？
```

用户确认后再进入 Step 2。不跳步。

**写作视角确认：**

| 文体 | 默认视角 |
|------|---------|
| 汇报材料 | 第一人称（"我单元""我司"→向领导汇报） |
| 工作总结 | 第一人称（"我部门""我单位"） |
| 述职报告 | 第一人称（"我"个人） |
| 调研报告 | 第三人称（调研组/课题组的客观发现） |
| 讲话稿 | 第一人称（讲话人本人） |
| 经验材料 | 第一人称（"我单位"分享经验） |
| 先进事迹 | 第三人称 |

### Step 2：生成大纲

**内建文体：** 用对应模板生成大纲 + 分配各部分字数。
**通用模板：** 根据头脑风暴结果构建大纲框架。

```
【角色】
你是体制内的资深笔杆子，擅长写各种单位材料，逻辑缜密，有着出色的文字功底、
丰富的政策理解和政治敏感性以及较强的信息整合与分析能力。

【任务】
我需要以[具体身份]写一篇[字数]的[材料类型]，请你根据【素材】和【大纲模板】
生成大纲，并分配好各部分字数。

【大纲模板】：
"""
[对应文体的大纲模板]
"""

【要求】
1. [根据文体特点提出具体要求，如"问题⇔对策一一对应"]

【素材】
"""
[粘贴原始素材]
"""
```

**常见大纲模板速查：**

| 文体 | 大纲结构 |
|------|---------|
| 工作总结 | 整体概述→主要做法(4点)→存在问题(4点)→工作计划(4点) |
| 述职报告 | 职责概述→履职情况(4点)→存在问题(4点)→下步打算(4点) |
| 汇报材料 | 背景→主要做法→存在问题→下步计划 |
| 讲话稿 | 开场→主体(3部分)→结尾 |
| 经验材料 | 背景→做法(3-4点)→成效→推广价值 |
| 调研报告 | 背景→现状→问题→建议 |
| 先进事迹 | 人物简介→事迹(3点)→升华 |
| 工作方案 | 总体目标→存在问题→主要措施→保障措施 |

### Step 3：逐一生成内容

⚠️ **核心原则：禁止一次性生成全部章节。** 每次最多生成 1-3 个连续段落，停下来让用户确认风格、内容和方向。

#### 分段投喂策略

每个文体的分段策略见对应模板。通用规则：

| 段落类型 | 投喂内容 | 目的 |
|---------|---------|------|
| **开头段**（概览/背景/人物简介） | 整个大纲 + 全部相关素材 | 让 AI 理解全文结构，写出恰当的开头 |
| **主体段**（做法/措施/事迹） | 仅投喂该部分的相关素材 | 让 AI 专注于内容深度，避免重复 |
| **问题段**（存在问题/不足） | 已生成的前面部分 + 问题素材 | 基于前文分析问题，确保逻辑连贯 |
| **对策/结尾段**（计划/建议/升华） | 已生成的前面全部内容 | 针对前文提对策，形成完整闭环 |

**① 开头段提示词**

"""

【要求】
1. 文风"短实新"，保持客观严谨
2. 句式工整，结构清晰，层次分明

4. 开头要有概括性，体现全文基调

丰富的政策理解和政治敏感性以及较强的信息整合与分析能力。

文体标题风格参考：工作总结→宏大+成效；述职报告→简洁+实绩；汇报材料→对仗+概括；
讲话稿→感召+有力度；经验材料→创新+可推广；调研报告→直面问题；先进事迹→精神内涵。

也可以参考第15-17章（拟定标题/生成结构化小标题/打磨金句）的具体提示词。

### Step 5：全文审查

写完初稿后，先做 self-review，再用 AI 修改。

**自审维度：**

| 维度 | 检查项 |
|------|--------|
| 结构完整 | 是否覆盖该文体所有必要章节 |
| 逻辑闭环 | 问题⇔对策是否一一对应 |
| 数据真实 | 有无编造数据（若有标 [待核查]） |
| 风格统一 | 全文语言风格、术语是否一致 |
| 格式统一 | 各段落格式（总-分、小标题、排比）是否一致 |

**用 AI 修改材料（三种改法，覆盖九成场景）：**

**① 整体提升** — 材料整体偏弱，需要全面升级

```
【角色】
请你扮演一个体制内的资深笔杆子。

【任务】
对提供的【材料】进行全面修改提升，在保持原意不变的基础上，
按照【要求】进行优化升级。

【要求】
1. 语言提升：使用规范的材料语言，用词准确、表达严谨
2. 逻辑优化：梳理文章结构，确保论证清晰、层次分明
3. 格式规范：统一标题层级、段落结构、标点使用
4. 重点突出：识别核心观点，合理安排详略
5. 政策融入：结合相关时政政策，提升政治站位
6. 完整输出：提供修改后的完整版本，并说明主要修改内容

【材料】
"""
[粘贴原文]
"""
```

**② 局部精修** — 整体框架没问题，某个具体问题需要解决

| 问题 | 核心指令 |
|------|---------|
| 内容太长啰嗦 | 压缩至XX字，保留核心信息，删除重复表述 |
| 开头平淡 | 重写开头，提供3个版本，要求开门见山 |
| 内容空洞 | 为每个观点补充具体数据或事例 |
| 逻辑混乱 | 按"总—分"结构重新梳理 |
| 表达生硬 | 优化句式节奏，长短句结合 |
| 结尾无力 | 重写结尾，提供2个版本 |

**③ 风格转换** — 受众或场合变化

说清楚"从什么风格"改成"什么风格"及关键转换要求。

---

### Step 6：去AI味

终稿完成后，加载 `humanizer` skill 清洗 AI 痕迹。

**两步操作：**

**① 清除 AI 默认格式**

```
请将以下【内容】转换为规范的公文格式：

【内容】
"""
[粘贴AI生成内容]
"""

【转换要求】
1. 删除所有Markdown格式标记（**加粗**、##标题、-列表、>引用等）
2. 将冒号列举改为自然表述（用"比如""例如""包括"等）
3. 将括号补充说明融入正文
4. 调整为规范的段落形式
5. 保持内容不变，只调整格式
```

**② 加载 humanizer skill 做全量检测**

运行 humanizer 的 29 种模式检测，重点清除：

- AI高频词汇（首先其次然而深刻深知切实有效综上所述等）
- 排比过度 / 三件套
- 空洞评价（"标志着""令人瞩目""为...奠定了基础"）
- 被动语态和主语缺失
- 假权威口吻（"业内专家认为""相关部门指出"）
- em dash 滥用 / emoji / 加粗滥用
- 协作式语气（"希望这对你有帮助""如你需要进一步了解"）

最终产出的人话版本与初稿一并给用户。

## 三、提示词工程技巧

### 角色扮演法

基础结构：设定角色（身份+背景+能力）→ 明确任务 → 提出要求

万能角色法：让 AI 自己决定角色（"你想让我扮演什么角色效果最好？"）

### 进阶玩法

**1. 提示词中加入方法论**
- 方式一：方法论写在【要求】模块内
- 方式二：方法论作为单独【方法论】模块，AI 先学会再执行

**2. 提示词中加入优质样例**
- 正例：让 AI 模仿优秀风格
- 反例：让 AI 规避常见问题
- 正反组合：效果最精准

**3. 提示词中投喂素材**
- 和"投喂优质样例"的区别：样例教"怎么写"，素材提供"写什么"

**4. 终极玩法：让 AI 生成自己的提示词**

```
请你扮演一个专家级的提示词生成器，你的任务是使用【提示词框架】来帮我生成
最优秀的提示词。我会和你说我要写什么内容，需要你先思考如何能超预期的写好
这个内容，再将你的答案提取重要的维度，写进【提示词框架】的"要求"中。

【提示词框架】
"""
【角色】
请你扮演一个xxx，你需要具备xxx能力和xxx特质

【任务】

【要求】
"""
```

AI 回复后再把你的所有具体需求发给它。

## 四、提示词格式规范

| 场景 | 推荐格式 |
|------|---------|
| 简单任务（通知/短函/便签） | 直接给指令即可 |
| 复杂任务（总结/报告/调研等） | 结构化提示词：【角色】【任务】【要求】【素材】 |
| 分隔长文本 | 三引号 """...""" 包裹素材 |
| 多层次结构 | 小节标题（【角色】【任务】） |
| 更清晰的层级 | Markdown 格式（# ## ### + 列表） |

## 六、底层原则

### 数据验证原则

写作中所有数字必须有可追溯的源数据：

| 原则 | 错误示例 | 正确做法 |
|------|---------|---------|
| 不编造百分比 | "占审计通报比重近50%"无出处 | 改为"该问题为央企审计中的高频问题" |
| 不推算无源数据 | "占总额12%"未提供总额 | 补充源数据后再计算 |
| 不写推测性结论 | 增加"附则纳入年度考核" | 保持真实输出 |

### 数据审计协议（结构化数据场景）

收到 Excel/CSV/表格类素材时：

1. 扫描全表 — 读取所有行，标注每个数据块的标题行和分类层级
2. 映射数据块 — 按标题区分（KPI目标、预算、实际执行），每块单独确认
3. 检查关系 — 跨块数据能否勾稽，不一致的标记⚠️待确认
4. 数字溯源 — 百分比必须有源数据可计算，无源数据改为定性表述
5. 逐一确认 — 将数据块结构+关系送给用户确认，得到确认后再写作

> **常见数据口径混淆：** 商机总量≠KPI目标≠已确认收入；总预算≠已使用资金≠未来预测

### 进阶模式：两步法（Feed + Prompt）

大量结构化数据时，拆成两步更可控：

**Step 1 — Feed（喂材料）：** 发原始数据 + "请先理解内容，不要输出任何分析，只需回复'已理解，请发送任务指令'即可"

**Step 2 — Prompt（发指令）：** AI 回复确认后，再发送结构化提示词

> 培训课件场景例外：教学示例中数据直接内嵌提示词（学员一次复制即用），讲师实际演示时再分步操作。

### 交付物质量标准（企业用户特化）

- 中文字体使用宋体（SimSun），正文 11pt（五号）
- `.docx` > `.txt` > `.md`，企业优先 Word 文档
- 示例提示词必须「复制即可用」——替换`[]`内占位符即可运行
- 案例素材用真实数据（具体条款编号、金额、日期、部门名），不得用占位符

### 写作伦理

- **不编造数据** — 素材没提供的一律标 [待核查]
- **不跳步** — 写完一段问用户是否调整
- **用户确认优先** — 大纲、素材、提示词调整都需要确认后执行
- **不写兜底承诺** — 汇报材料是分析现状、提建议，不是替领导下结论
- **不写"偏差/超支"口吻，写"调整为"** — 数据变动是正常调整，不贴负面标签
- **问题与对策必须成对出现** — 每个风险点后跟行动项
- **数字勾稽检查** — 全文数字做交叉验证（预算=已用+预测+剩余）
