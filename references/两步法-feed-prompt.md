# 两步法（Feed + Prompt）— AI交互工作流

适用于需要AI处理大量原始数据再执行特定任务的场景，如审计底稿编写、数据分类分析、报告生成等。

---

## 两步法流程

### 第1步 — Feed（喂材料）

将原始数据发送给AI，附确认指令防止提前输出：

```
这是本次审计的原始检查记录，请先理解材料内容，不要输出任何分析，
只需回复"已理解，请发送任务指令"即可。
```

确认指令的作用：
- 防止AI提前分析跑偏
- 确认数据完整到达（没有截断）
- 分离"数据接收"和"任务执行"两个阶段

### 第2步 — Prompt（发指令）

AI回复确认后，再单独发送结构化提示词：

```
【角色】你是[具体角色]，熟悉[领域]业务
【输出要求】
1. [格式约束]
2. [内容要点]
3. [字数限制]
```

---

## 优势

- 同一份原始数据可搭配不同指令任务（先底稿、再整改通知），无需重复粘贴
- 避免AI在未收到完整指令时提前输出不完整结果
- 每次任务变更只需替换指令部分，数据不动

---

## 注意事项

- Feed阶段的数据必须完整，不含角色/任务要求
- Feed指令中的"不要输出分析"要写清楚，否则部分AI仍会开始分析
- 数据量大的场景（几十条记录以上），建议分批投喂或上传文件
