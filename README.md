# SOP Writing — AI 辅助公文写作工作流

基于《AI 写材料知识体系》的标准化写作流程，将 AI 辅助公文写作从"问一句答一句"升级为可复用的六步 SOP。

## 概述

```mermaid
flowchart LR
    A[Step 1<br/>拆解+头脑风暴] --> B[Step 2<br/>生成大纲]
    B --> C[Step 3<br/>逐一生成]
    C --> D[Step 4<br/>生成标题]
    D --> E[Step 5<br/>全文审查]
    E --> F[Step 6<br/>去AI味]
```

**六步流程：** 拆解写作任务 → 生成大纲 → 逐段生成内容 → 生成标题 → 全文审查 → 去AI味

**提示词结构：** 【角色】【任务】【要求】【素材】

## 特色

- **11 种文体模板** — 覆盖工作总结、述职报告、汇报材料、讲话稿、经验材料、调研报告、先进典型事迹、工作方案、会议纪要（3 子类型）、新闻稿（4 子类型）+ 通用模板
- **分段投喂策略** — 每一段都有专属提示词，不是一条提示词走到底
- **去 AI 味** — 集成 humanizer 的 29 种 AI 痕迹检测
- **数据审计协议** — Excel/表格类素材的规范化处理流程
- **两步法（Feed + Prompt）** — 大量结构化数据的处理工作流

## 快速开始

### Step 1：确认文体 + 收集素材

确定要写的文体类型，让用户提供素材。有范文更好。

### Step 2：头脑风暴

按四个维度与用户沟通：

| 维度 | 内容 |
|------|------|
| 写作背景与用途 | 给谁看？什么场合？直接原因？ |
| 核心信息 | 想让读者记住什么？难点/敏感点？ |
| 作者与视角 | 以谁的身份写？语气？ |
| 风格偏好 | 喜欢的范文？不想要的风格？字数？ |

### Step 3-6：按六步流程执行

见 `SKILL.md` 中各步骤的具体提示词。

## 去 AI 味

Step 6 加载 [humanizer](https://github.com/blader/humanizer) 做全量检测，重点清除：

- AI 高频词汇（首先、其次、然而、深刻、深知、综上所述等）
- 排比过度 / 三件套
- 空洞评价（"标志着""令人瞩目""为……奠定了基础"）
- 被动语态和主语缺失
- 假权威口吻（"业内专家认为""相关部门指出"）
- 协作式语气（"希望这对你有帮助"）

## 核心原则

- **不编造数据** — 素材没提供的一律标 [待核查]
- **不跳步** — 写完一段问用户是否调整
- **不写兜底承诺** — 分析现状提建议，不是替领导下结论
- **问题与对策必须成对出现**
- **数字勾稽检查** — 预算 = 已用 + 预测 + 剩余

## 数据审计（Excel/表格场景）

结构化数据必须走五步审计流程：扫描全表 → 映射数据块 → 检查关系 → 数字溯源 → 逐一确认。

常见口径混淆：商机总量 ≠ KPI 目标 ≠ 已确认收入；总预算 ≠ 已使用资金 ≠ 未来预测。
